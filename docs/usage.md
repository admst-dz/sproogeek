# Руководство пользователя

## Назначение

Spruzhyk Configurator позволяет клиенту собрать изделие в 3D-конфигураторе, сохранить заказ в базе данных и отслеживать его статус. Сотрудники с ролями `dealer`, `admin` и `owner` работают с заказами через служебную панель. Администраторы дополнительно получают доступ к JSON-файлам типов заказов.

## Роли

| Роль | Возможности |
| --- | --- |
| `client` | оформление заказа, просмотр своих заказов, выбор подроли клиента |
| `dealer` | управление своими продуктами и заказами, явно привязанными к дилеру |
| `admin` | просмотр всей базы заказов, редактирование JSON типов заказов |
| `owner` | те же права, что у администратора |

Регистрация через публичный API всегда создает клиента. Роли `dealer`, `admin` и `owner` должны назначаться доверенным способом на стороне базы данных или внутреннего административного процесса.

## Основной сценарий клиента

1. Пользователь открывает сайт.
2. Выбирает изделие и параметры в конфигураторе.
3. Добавляет конфигурацию в корзину.
4. Авторизуется или регистрируется.
5. Заполняет контактные данные.
6. Подтверждает заказ.
7. Заказ сохраняется в таблице `orders`.
8. Backend запрашивает render у контейнера `renderer` и сохраняет изображение в `uploads/renders`.
9. Backend публикует событие в Kafka, если Kafka доступна.
10. Пользователь видит заказ в личном кабинете.

## Сценарий администратора

1. Администратор входит в учетную запись с ролью `admin` или `owner`.
2. Приложение открывает служебную панель.
3. Вкладка заказов загружает данные через `/api/v1/admin/orders`.
4. Вкладка JSON типов заказов загружает список файлов через `/api/v1/admin/order-types`.
5. При выборе файла приложение показывает его содержимое в JSON-редакторе.
6. После сохранения backend валидирует JSON, записывает файл атомарно и фиксирует событие в CSV-логах.

## Логи

Сервер создает папку `logs` в корне проекта. По умолчанию используются CSV-файлы вида:

```text
logs/events_YYYYMMDD_0001.csv
logs/events_YYYYMMDD_0002.csv
```

Один файл не превышает 10 000 строк, включая заголовок. После достижения лимита создается следующий файл с увеличенным индексом. Логи содержат:

| Поле | Смысл |
| --- | --- |
| `timestamp` | время события в UTC |
| `event_type` | тип события, например `HTTP_REQUEST`, `ORDER_CREATED`, `KAFKA_MESSAGE_SENT` |
| `direction` | направление взаимодействия: `user->backend`, `backend->renderer`, `backend->kafka` |
| `actor_type`, `actor_id`, `actor_email` | роль и идентификаторы пользователя, если доступны |
| `method`, `path`, `status_code`, `latency_ms` | HTTP-контекст |
| `entity_type`, `entity_id` | объект события: заказ, продукт, JSON-файл |
| `details_json` | дополнительные безопасные детали в JSON |

Пароли, токены и OAuth-коды в query-параметрах маскируются.

## Переменные окружения

| Переменная | Назначение |
| --- | --- |
| `DATABASE_URL` | строка подключения SQLAlchemy async |
| `SECRET_KEY` | ключ подписи JWT, обязателен в production |
| `APP_ENV` или `ENVIRONMENT` | окружение: `development`, `production` |
| `ACCESS_TOKEN_EXPIRE_MINUTES` | срок жизни JWT, по умолчанию 43 200 минут |
| `ALLOWED_ORIGINS` | список CORS origin через запятую |
| `ALLOWED_HOSTS` | список разрешенных Host-заголовков |
| `EVENT_LOG_DIR` | кастомная директория логов |
| `EVENT_LOG_MAX_LINES` | лимит строк в одном CSV-файле |
| `ORDER_TYPES_DIR` | кастомная директория JSON типов заказов |
| `ORDER_TYPE_FILE_MAX_BYTES` | максимальный размер одного JSON-файла |
| `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET` | OAuth Google |
| `YANDEX_CLIENT_ID`, `YANDEX_CLIENT_SECRET` | OAuth Yandex ID |
| `VK_CLIENT_ID`, `VK_CLIENT_SECRET` | OAuth VK |
| `KAFKA_BOOTSTRAP_SERVERS` | адрес Kafka |

## Безопасная эксплуатация

- В production обязательно задайте `SECRET_KEY`.
- Настройте `ALLOWED_ORIGINS` под реальные домены сайта.
- Не храните реальные секреты в Git.
- Выдавайте роли `admin`, `owner`, `dealer` только через доверенный внутренний процесс.
- Регулярно выгружайте и архивируйте `logs`, если сервер работает под высокой нагрузкой.
- Для загрузки логотипов разрешены только PNG, JPEG и WEBP; SVG/EPS не принимаются из-за риска активного содержимого.
